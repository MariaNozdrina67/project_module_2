# Анализ поведения пользователей Яндекс Афиши

## Описание проекта
Исследование факторов, влияющих на повторные заказы пользователей сервиса Яндекс Афиша. На основе данных о покупках, мероприятиях и географических характеристиках построены профили пользователей, проведён сегментный анализ, выявлены ключевые драйверы лояльности и даны рекомендации для маркетинговой команды.

## Данные
- База данных: `data-analyst-afisha` (схема `afisha`), таблицы `purchases`, `events`, `city`, `regions`.
- Курсы валют: внешний датасет `final_tickets_tenge_df.csv`.

## Этапы работы
1. Выгрузка и предобработка данных (приведение выручки к рублям, фильтрация выбросов).
2. Создание профилей пользователей (агрегация по `user_id`).
3. Исследовательский анализ:
   - распределение по сегментам (мероприятие, устройство, регион, оператор);
   - анализ возвратов (доля повторных покупок);
   - влияние среднего чека, количества билетов, дня недели и интервала между заказами;
   - корреляционный анализ (`phi_k`).
4. Формулировка выводов и рекомендаций.

## Использованные библиотеки
- pandas, numpy, matplotlib, seaborn, sqlalchemy, psycopg2-binary, phik, python-dotenv.

## Как запустить проект
1. Клонируйте репозиторий: `git clone ...`
2. Установите зависимости: `pip install -r requirements.txt`
3. Создайте файл `.env` с параметрами подключения к БД
4. Запустите ноутбук `project_module_2.ipynb`.

## Основные выводы
- 61.7% пользователей совершают 2+ заказа, 28.9% – 5+ заказов.
- Наиболее значимый фактор – средний интервал между покупками: чем он короче, тем выше число заказов.
- Основные точки входа: концерты, мобильные устройства, Каменевский регион, оператор «Билеты без проблем».
- Самый высокий возврат – у пользователей с 2–3 билетами (74%).
- День недели первой покупки не влияет на возврат.
- Рекомендуется стимулировать частоту возвратов и работать с сегментом 2–4 заказа для перевода в суперактивные (5+).
